<objective>
Build the **Native Uninstaller Driver**: given an `InstalledProgram`, parse its `UninstallString`/`QuietUninstallString`, execute the underlying uninstaller as a child process attached to a Win32 Job Object (for tree-kill on cancel), stream live stdout/stderr to the UI via `IProgress<string>`, and surface exit code + timing as a structured outcome.

Purpose: This is the execution engine of the wizard's "Run native uninstaller" step. Without correct argument parsing and cancellation discipline, the whole feature is unsafe (zombie installer processes, corrupted state).
Output: `INativeUninstallerDriver` + parser + Job Object child process executor + tests.
</objective>

<reference_implementation>
BCUninstaller's `BulkCrapUninstaller.Functions.Uninstaller.UninstallManager` performs equivalent work. Key takeaways (do NOT copy code — read for patterns):
- MSI: detect by checking if executable path ends with `MsiExec.exe` (case-insensitive). If the args contain `/I{guid}` (an Install string), replace with `/X{guid}`. Append `/qn /norestart` for silent.
- InnoSetup: detect by sentinel arg `_?=` in the uninstall string — preserve it (Inno requires this), append `/SILENT /SUPPRESSMSGBOXES /NORESTART`.
- NSIS: typically already supports `/S` — append `/S` if silent.
- Otherwise: rely solely on `QuietUninstallString` for silent; do NOT guess flags.

- `CreateJobObject` -> `SetInformationJobObject(JobObjectExtendedLimitInformation, JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE)` -> `AssignProcessToJobObject` BEFORE the child process is allowed to spawn descendants.

    Implement RunAsync algorithm:

    1. Validate executable path: `if (!File.Exists(command.ExecutablePath)) return new UninstallOutcome(UninstallStatus.ExecutionFailure, null, TimeSpan.Zero, 0, $"Not found: {command.ExecutablePath}");`
    2. Create Job Object: `var hJob = CreateJobObject(IntPtr.Zero, null); if (hJob == IntPtr.Zero) return new UninstallOutcome(ExecutionFailure, ...);`
    3. Set KILL_ON_JOB_CLOSE: marshal a `JOBOBJECT_EXTENDED_LIMIT_INFORMATION { BasicLimitInformation = { LimitFlags = JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE } }` and call `SetInformationJobObject(hJob, JobObjectExtendedLimitInformation, ptr, (uint)Marshal.SizeOf<...>())`.
    4. Build `ProcessStartInfo`:
       ```
       FileName = command.ExecutablePath,
       Arguments = command.Arguments,
       WorkingDirectory = command.WorkingDirectory ?? "",
       UseShellExecute = false,
       RedirectStandardOutput = true,
       RedirectStandardError = true,
       CreateNoWindow = command.IsSilent
       ```
    5. `using var process = new Process { StartInfo = psi };`
       Subscribe `process.OutputDataReceived` and `process.ErrorDataReceived` -> if `e.Data != null`, `Interlocked.Increment(ref lineCount); output?.Report(e.Data);`

    7. `process.BeginOutputReadLine(); process.BeginErrorReadLine();`

    11. Normal exit: read `process.ExitCode`. Return `Success` if 0, else `NonZeroExit`.
