<objective>
Build the **Publisher Rule Engine**: a JSON-driven engine that knows specific patterns of residues per publisher (Adobe, Autodesk, JetBrains, Mozilla, Microsoft, Steam, Epic, ...). Rules are embedded in the exe + extensible via `%LocalAppData%\DiskScout\publisher-rules\*.json`. The wizard merges rule-derived paths into the residue list to bootstrap deep cleanup with publisher-specific knowledge.

Purpose: This is what makes Plan 03's residue scanner Revo-Pro-grade. Generic fuzzy matching alone misses patterns like Adobe's `Adobe ARM`, JetBrains's `.IntelliJIdea2024.3` config dirs, etc.
Output: `IPublisherRuleEngine` + embedded rule JSONs + tests.
</objective>

    public sealed record PublisherRule(

    Implementation:

    1. `LoadAsync`:
       - Step A: enumerate embedded resources via `_resourceAssembly.GetManifestResourceNames()`. Filter to names starting with `"DiskScout.Resources.PublisherRules."` and ending with `.json`. For each, `using var stream = _resourceAssembly.GetManifestResourceStream(name);` deserialize via `JsonSerializer.DeserializeAsync<PublisherRule>(stream, options)` with `JsonSerializerOptions { PropertyNameCaseInsensitive = true }`. On JsonException, `_logger.Warning(ex, "Bad embedded rule {Name}", name);` and continue.
       - Step B: if `Directory.Exists(_userRulesFolder)`, enumerate `*.json` and parse the same way. Insert into the result with last-write-wins semantics: Dictionary<string,PublisherRule> keyed by `Id` — user rules overwrite embedded ones.
       - Set `AllRules` to `Dictionary.Values.ToList().AsReadOnly()`.
       - `_logger.Information("Loaded {N} publisher rules ({Embedded} embedded, {User} user)", count, embeddedCount, userCount);`

    2. `Match(publisher, displayName)`:
       - For each rule, evaluate:
         - `publisherMatch = !string.IsNullOrEmpty(publisher) && Regex.IsMatch(publisher, rule.PublisherPattern, RegexOptions.IgnoreCase, TimeSpan.FromMilliseconds(200))`
         - `displayMatch = string.IsNullOrEmpty(rule.DisplayNamePattern) || Regex.IsMatch(displayName ?? "", rule.DisplayNamePattern, ...)`
         - Skip if `!publisherMatch || !displayMatch`.
         - Score: `int score = 10; if (!string.IsNullOrEmpty(rule.DisplayNamePattern)) score += 100; // DisplayName-narrowed = much more specific`
       - Wrap each Regex evaluation in try/catch (RegexMatchTimeoutException, ArgumentException) — on bad regex, log warning + skip rule.
       - Return matches sorted by SpecificityScore desc.

    3. `ExpandTokens(template, publisher, displayName)`:
       - Replace each `%VAR%` with `Environment.ExpandEnvironmentVariables` for cross-compat (it natively handles `%LocalAppData%`, `%ProgramFiles(x86)%`, etc.).
       - Then replace `{Publisher}` and `{DisplayName}` literal tokens with the supplied values (skip if values null).
       - If a result still contains a literal `%...%` after expansion (env var unknown), log Debug and return the unexpanded string.
