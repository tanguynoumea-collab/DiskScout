<objective>
Extract `IServiceEnumerator` and `IScheduledTaskEnumerator` from their current `internal` location inside `Services/ResidueScanner.cs` into dedicated public files (`Services/IServiceEnumerator.cs` and `Services/IScheduledTaskEnumerator.cs`), add two new public interfaces `IDriverEnumerator` and `IAppxEnumerator` with production implementations, and create the `MachineSnapshot` model + `IMachineSnapshotProvider` with a lazy TTL-5min cache that populates all 4 enumerators in parallel.

Purpose: Plan 10-04's `MultiSourceMatcher` (pipeline step [4]) needs a single fast lookup against an indexed snapshot of the machine state. `Get-AppxPackage` and `pnputil /enum-drivers` are slow (1-3s); without caching, every orphan candidate would re-enumerate the entire machine. The TTL allows the user to launch a scan, then re-scan after 4 minutes without paying the indexation cost twice.
Output: 2 promoted interfaces + 2 new interfaces + 2 production enumerator impls + 1 model + 1 provider + ~14 new tests. Zero regression on the existing 130 tests (the 46 ResidueScanner tests in particular MUST continue to pass after the interface refactor).
</objective>

```csharp
namespace DiskScout.Models;

public sealed record MachineSnapshot(
    DateTime CapturedUtc,
    IReadOnlyList<ServiceEntry> Services,
    IReadOnlyList<DriverEntry> Drivers,
    IReadOnlyList<AppxEntry> AppxPackages,
    IReadOnlyList<ScheduledTaskEntry> ScheduledTasks)

    public IReadOnlySet<string> DriverProviderTokens { get; init; } = new HashSet<string>(StringComparer.OrdinalIgnoreCase);

}

public sealed record ServiceEntry(string Name, string DisplayName, string? BinaryPath);
public sealed record DriverEntry(string PublishedName, string? OriginalFileName, string? Provider, string? ClassName);
public sealed record AppxEntry(string PackageFullName, string? PackageFamilyName, string? Publisher, string? InstallLocation);
public sealed record ScheduledTaskEntry(string TaskPath, string? Author, string? ActionPath);
```

IMachineSnapshotProvider.cs (NEW):

```csharp
namespace DiskScout.Services;
using DiskScout.Models;

public interface IMachineSnapshotProvider
{
    /// <summary>Returns the cached snapshot if still fresh (age < TTL), else builds a new one
    /// by running all 4 enumerators in parallel via Task.WhenAll. Thread-safe (re-entrant calls
    /// share the same in-flight Task). Default TTL 5 min.</summary>
    Task<MachineSnapshot> GetAsync(CancellationToken cancellationToken = default);
    void Invalidate(); // forces next GetAsync to rebuild
}
```

    - `MachineSnapshot` is an immutable record holding 4 entry lists + 3 indexed `IReadOnlySet` lookup helpers (`ServiceBinaryPathPrefixes`, `DriverProviderTokens`, `AppxInstallLocationPrefixes`). The 3 indexes are constructed inline at snapshot-build time — NOT in a separate pass. Once built, `MachineSnapshot` is fully immutable. The 3 entry types `ServiceEntry`/`DriverEntry`/`AppxEntry`/`ScheduledTaskEntry` are simple records.
    - `MachineSnapshotProvider`'s constructor takes `ILogger logger, IServiceEnumerator services, IDriverEnumerator drivers, IAppxEnumerator appx, IScheduledTaskEnumerator tasks, TimeSpan? ttl = null`. Default TTL is 5 minutes. The TTL is stored in a private readonly field.
    - `GetAsync` is thread-safe and re-entrant. Logic: lock-protected read of `_cached`. If `_cached != null && (UtcNow - _cached.CapturedUtc) < _ttl`, return `Task.FromResult(_cached)`. Else, ensure exactly one Task is running via `Lazy<Task<MachineSnapshot>>` pattern (or `_inFlight` Task field with double-check lock). The build runs all 4 enumerators in parallel via `Task.WhenAll` of `Task.Run(() => services.EnumerateServices().Select(...).ToList())` etc. After all 4 complete, the snapshot record is constructed, the 3 indexes computed, then `_cached` is set and returned.
    - `Invalidate()` simply nulls `_cached` and `_inFlight`. Next `GetAsync` rebuilds.
    - `CancellationToken` flowed through: each `Task.Run` checks `ct.ThrowIfCancellationRequested` once before starting (the underlying enumerators are synchronous IO and cannot themselves be cancelled mid-stream — accepted limitation for this phase).
    - The 3 index sets are populated as: `ServiceBinaryPathPrefixes = services.Where(s => s.BinaryPath != null).Select(s => Path.GetDirectoryName(s.BinaryPath)).Where(p => !string.IsNullOrEmpty(p)).ToHashSet(OrdinalIgnoreCase)`. Similar for `AppxInstallLocationPrefixes` (each entry's `InstallLocation` if non-null/non-empty). `DriverProviderTokens` collects each driver's `Provider` token (split on whitespace, lowercased).
